1、创建秘钥 (之后再本机上与github自己的账号连接就不用一直输入账号密码了)

git ssh-keygen -t rsa -C "github里注册的邮箱"

输入以上命令一直按回车后，会生成.ssh文件夹，进去，打开id_rsa.pub,复制里面的key

回到github上，进入 Account Settings（账户配置），左边选择SSH Keys，Add SSH Key,title随便填，粘贴在你电脑上生成的key。

为了验证是否成功，输入以下命令：

$ ssh -T git@github.com

回车提示为“Hi [自己的用户名]! You've successfully authenticated, but GitHub does not provide shell access.”那就说明我们已经成功连上了github

2、连接github里的某个库 上传本地库到github

git remote add [自己取一个名字] git@github.com:[github上的用户名]/[库名]

回车后就可以连上这个库了（取一个名字用于替换后边这个“git@github.com:[github上的用户名]/[库名]”）方便以后命令书写

git push -u [刚刚自己取得名字] [要上传到github上的哪个分支名]

回车就可以 把本地创建的库文件对应上传到github里自己的对应的库了

2、更新github库到本地库

git fetch

回车后就可以从远程仓库下载新分支与数据

git merge [上边自己取得名字]/[库名]

回车就可以 从远端仓库提取数据并尝试合并到当前分支

2、git命令

git remote(查看当前配置有哪些远程仓库)

git remote -v(执行时加上 -v 参数，你还可以看到每个别名的实际链接地址。)

git fetch(从远程仓库下载新分支与数据)

git pull(从远端仓库提取数据并尝试合并到当前分支)

或

git fetch [alias](从远程仓库下载新分支与数据 alias为自己取得别名)

git merge [alias]/[branch] (从远端仓库提取数据并尝试合并到当前分支)

git push [alias] [branch] （推送你的新分支与数据到某个远端仓库）注：命令将你的 [branch] 分支推送成为 [alias] 远程仓库上的 [branch] 分支

git push -u [alias] [branch] （推送你当前分支到远端仓库branch分支）

git remote rm [别名]（删除远程仓库）

git status（查看提交状态）

git status -s（查看提交状态简略）

touch [文件名]（新建一个文件）

vim [文件名]（进入文件编辑）

a || ZQ || ZZ（激活编辑器输入 || 不保存退出文件编辑 || 保存之后退出文件编辑）

git add . || git add [文件名 文件名...]（提交所有本地文件但暂存区 || 提交指定的文件到暂存区）

git commit -m "[输入提交注释]"（把暂存区的文件提交到git版本库）

git checkout . || git checkout --[文件名]（会用暂存区全部的文件替换工作区的文件 || 会用暂存区指定的文件替换工作区的文件）

git rm --cached [文件名]（会直接从暂存区删除文件，工作区则不做出改变）

git reset (HEAD || master)（暂存区的目录树会被重写，被 master 分支指向的目录树所替换，但是工作区不受影响）

git checkout HEAD . || git checkout HEAD [文件名]（会用 HEAD 指向的 master 分支中的全部或者部分文件替换暂存区和以及工作区中的文件）
